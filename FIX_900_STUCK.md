# 修复：900 条评论卡住问题

## 问题诊断

爬虫在 **893-898 条评论**处卡住，表现特征：
- 连续多轮没有新评论加载
- 滚动没有进度（remaining=0 不变）
- 无法触发新的评论加载

## 根本原因分析

### 缺失的 500-1000 区间策略
原始优化只覆盖了：
- < 500：normal mouse scroll
- 1000+：激进 JS 滚动 + 长等待

**但 500-1000 区间缺乏专门策略**！这导致：
1. 在 900 条时没有足够的激进滚动
2. 没有针对这个区间的长等待机制
3. 无法唤醒 Google 加载新评论

### 等待阈值不对
- 原来 1000+ 时才降低阈值为 2
- 500+ 时仍然是 3，导致卡顿

## 实施的修复

### ✅ 修复 1：500-1000 专门策略

```javascript
// 新增完整的 500-1000 处理逻辑
if (totalReviews >= 500 && totalReviews < 1000) {
  // 每 30 轮：激进唤醒 (step: 1600, times: 7)
  // 每 15 轮：中等唤醒 (step: 1400, times: 6)
  // 默认：标准滚动 (step: 1200, times: 5)
}
```

效果：
- 每 15 轮获得"中等唤醒"
- 每 30 轮获得"激进唤醒"
- 远比之前只用 mouse 的方式更有效

### ✅ 修复 2：更频繁的长等待

```javascript
// 500-1000 区间
if (reviewMap.size >= 500 && reviewMap.size < 1000 && stableCount % 15 === 0) {
  await randomDelay(45000, 60000);  // 45-60 秒长等待
}

// 1000+ 区间改进
if (reviewMap.size >= 1000 && stableCount % 30 === 0) {
  await randomDelay(60000, 75000);  // 60-75 秒
}
```

这给 Google 足够的时间加载新评论。

### ✅ 修复 3：强制唤醒机制

```javascript
if (stableCount >= 10 && reviewMap.size >= 500) {
  // 执行超激进滚动：step 2500, times 10
  // 然后等待 10 秒
  // 最后等待 8-12 秒
}
```

当连续 10 轮没有新评论时，触发"暴力唤醒"。

### ✅ 修复 4：降低稳定性阈值

```javascript
let waitThreshold = 3;
if (reviewMap.size >= 2000) waitThreshold = 2;
if (reviewMap.size >= 1000) waitThreshold = 2;
if (reviewMap.size >= 500) waitThreshold = 2;  // ← 新增
```

500+ 条时更容易触发等待机制。

## 技术细节

### 滚动参数说明

| 名称 | step | times | delay | 效果 |
|-----|------|-------|-------|------|
| 超激进 | 2500 | 10 | 1500 | 一次性大幅滚动 |
| 激进 | 1600-1800 | 7 | 1100-1200 | 频繁快速滚动 |
| 中等 | 1400-1500 | 6 | 1000 | 均衡策略 |
| 标准 | 1200-1300 | 5-6 | 900-1000 | 持续滚动 |

- `step`：每次滚动的像素数
- `times`：连续滚动的次数
- `delay`：每次滚动间隔（毫秒）

### 等待机制

- **noMoveCount >= 2**（500+）：滚动 2 次无进度 → 等待 10-15 秒
- **stableCount % 15**（500-1000）：每 15 轮等待 45-60 秒
- **stableCount % 30**（1000+）：每 30 轮等待 60-75 秒
- **stableCount >= 10**：强制唤醒超激进滚动
