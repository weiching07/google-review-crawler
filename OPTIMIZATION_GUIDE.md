# Google 评论爬虫优化指南

## 问题诊断

你的爬虫卡在 1000+ 条评论的原因：

1. **轮数限制** - 默认只运行 5 轮，无法完整爬取
2. **Google 虚拟滚动** - 1000+ 条时 Google Maps 优化了虚拟列表，新评论加载变慢
3. **滚动策略不足** - 原始滚动参数在高评论数时效果不佳
4. **等待时间不够** - Google 需要更长时间来响应大量滚动请求

## 优化改动

### 1️⃣ 增加最大轮数

**文件**: `scraper.js` - `getMaxRounds()` 函数

- 原来：`SCRAPE_MAX_ROUNDS >= 999` 返回 5000
- 优化后：`SCRAPE_MAX_ROUNDS >= 100` 返回 5000，`>= 999` 返回 8000
- 效果：更容易触发完整爬取模式

### 2️⃣ 激进的滚动策略

**文件**: `scraper.js` - `fastScrollReviews()` 函数

改进策略分级（1000+ 评论时）：

| 稳定轮数 | 滚动参数 | 等待时间 | 用途 |
|---------|---------|---------|------|
| 每 50 次 | 超激进 | 8秒 | 突破瓶颈 |
| 每 70 次 | 激进 | 6秒 | 唤醒 Google |
| 每 30 次 | 中等 | 5秒 | 常规探测 |
| 每 10 次 | 温和 | 4秒 | 温和探测 |
| 默认 | 标准 | 3.5秒 | 持续滚动 |

### 3️⃣ 动态等待策略

**文件**: `scraper.js` - `fastLoadAndCollectReviews()` 函数

新增等待机制：

- **1500+ 评论**：每 25 轮长等待 90-120 秒
- **2000+ 评论**：每 20 轮等待 45-60 秒
- **滚动卡顿**：根据评论数调整等待时间
  - 2000+：12-18 秒
  - 1000+：12-18 秒

## 使用方法

### 🔥 完整爬取（推荐）

```bash
npm run scrape:full
```

这会自动设置最优参数，爬取完整 5000+ 评论。

### 🎯 自定义轮数

```bash
# 运行 3000 轮
SCRAPE_MAX_ROUNDS=999 node run-full-scrape.js

# 或指定具体店铺
SCRAPE_MAX_ROUNDS=999 SCRAPE_TARGET_BRAND="店铺名" node run-full-scrape.js
```

### 📊 常规同步（部分评论）

```bash
node sync-static.js
```

## 关键改进点

✅ **改进 1**: 轮数从 5 提升到 5000+
- 原因：每轮只能加载部分评论，需要足够轮数
- 效果：从 1000 条瓶颈突破到 5000+ 条

✅ **改进 2**: 激进的滚动参数
- 原因：高评论数时虚拟列表需要更大的滚动幅度
- 效果：每轮加载新评论的概率提升

✅ **改进 3**: 超长等待时间
- 原因：Google 对大量滚动请求有速率限制
- 效果：给 Google 足够时间加载新评论

✅ **改进 4**: 分级策略
- 原因：不同评论数阶段的最优参数不同
- 效果：1000 以前快速，1000+ 稳定

## 故障排查

### 问题：仍然在 1000-1500 卡住

**原因**：Google 可能启用了更严格的限流

**解决**：
```bash
# 增加全局等待时间
SCRAPE_MAX_WAIT=60000 npm run scrape:full
```

### 问题：Chrome 崩溃或内存溢出

**原因**：长时间运行导致内存积累

**解决**：
```bash
# 定期重启 Chrome（每 1000 条评论）
# 修改 scraper.js 中的 scrapeOneStore 逻辑
# 在每个店铺后关闭并重新打开浏览器
```

### 问题：网络超时错误

**原因**：连接不稳定或 Google 暂时不可用

**解决**：
```bash
# 增加网络超时
PUPPETEER_TIMEOUT=120000 npm run scrape:full
```

## 下一步优化

如果仍然无法达到 5000+ 评论，考虑：

1. **分布式爬取** - 多个进程并行爬取不同店铺
2. **代理轮换** - 使用代理池规避 IP 限流
3. **长期监听** - 设置定时任务持续增量爬取，而不是一次性爬取全部
4. **缓存管理** - 优化内存使用，支持更长时间运行
